# Task 6 — Watermarking

**Watermarking**: Optional subtle watermark in the bottom corner, toggleable. This needs a little bit of thought from a UX point of view, since the only reason to add a watermark to a video is so it's present on any downloaded/streamed videos. This would mean adding it to source.mp4 and any video derivitives, but ALSO I'll probably want to retain an unwatermarked version of source.mp4 too. The obvious best option here is to simply create a new watermarked derivitive source.mp4 and generate the lowe-res derivitives from that. Oh, but the obvious downside to this is disc storage, basically. Now while it would obviously be easier to add the watermark as part of the recording composition locally, That's probably not a sensible option because it wouldn't apply to uploaded recordings and also we sould not have a clean watermarkless source.mp4 on the server. One thing that's also worth considering here is the fact that portions of a video in screen+cam mode can have the PiP overlay in any one of the four corners, so while a watermark might usually be in the bottom left, we'd probs want to swap it to the bottom right whenever the PiP is in the bottom left. This should be doable based on the data that we have on the server in recordings..json. Now having said all that, it might well be that this simply isn't worth doing at all. Or perhaps And we have a toggle on the video admin page to switch these on and we simply overlay a watermark on the web video players (ie not the videos themselves), and only generate new videos with baked-in watermarks when someone first hits the /:slig/raw endpoint for the first time? But then there are obvious downsides to that as well.
